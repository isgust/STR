Como Rodar o Projeto

O projeto é construído em Go (Golang) com uma interface em HTML, CSS e JS puros. Para executá-lo em sua máquina local, siga os passos:

  • Certifique-se de ter o Go (versão 1.18+) instalado na sua máquina.
  • Abra o terminal na raiz do projeto (pasta onde estão main.go e game.go).
  • Execute o comando abaixo para iniciar o servidor:
    go run .
  • Abra seu navegador favorito e acesse http://localhost:8080.

O Conceito de Tempo Real e Latência

Quando jogamos na internet, a informação viaja do nosso computador até o servidor do jogo. Esse tempo de viagem é conhecido como Latência ou Ping.

O Problema Sem Compensação

Se um jogador tem 400ms de latência, qualquer ação que ele fizer levará quase meio segundo para chegar ao servidor. No Super Latency Bomber, o objetivo é apertar o botão exatamente no tempo limite. Se o servidor ignorar a latência, o jogador com internet mais lenta será penalizado injustamente, pois seu clique sempre chegará atrasado.

A Solução Lag Compensation

Para criar um ambiente competitivo justo, os servidores modernos utilizam a Compensação de Lag (Lag Compensation). O servidor usa o Ping do jogador para calcular o exato momento em que a ação ocorreu na tela do cliente, essencialmente "voltando no tempo".

Veja no código abaixo como implementamos isso na função handlePress em game.go:

func (g *Game) handlePress(player int, simulatedLatency int64) {
    // Atraso artificial para simular a rede (Latência)
    time.Sleep(time.Duration(simulatedLatency) * time.Millisecond)

    serverReceiveTime := time.Now().UnixMilli()
    var eventTime int64

    if g.lagComp {
        // Compensação de Lag ATIVADA
        // O servidor deduz a latência para saber quando ocorreu o clique real
        eventTime = serverReceiveTime - simulatedLatency
    } else {
        // Compensação de Lag DESATIVADA
        // O servidor julga com base na hora de chegada. Injusto!
        eventTime = serverReceiveTime
    }

    // Validação do clique em relação ao momento da explosão
    if eventTime >= g.explosionTime {
        status = "DEAD"
    } else {
        status = "ALIVE"
    }
}

Arquitetura do Projeto

O sistema foi construído pensando na simplicidade e eficiência dos WebSockets.

  • main.go: Responsável por inicializar o servidor HTTP e fazer o Upgrade das conexões para WebSocket.
  • game.go: Mantém a Game Loop e as variáveis de estado do servidor, recebendo as ações dos jogadores e emitindo o estado de volta para todos os clientes em tempo real (Broadcasting).
  • main.js: Envia mensagens JSON curtas via ws.send() e renderiza o relógio a 60FPS usando requestAnimationFrame.
